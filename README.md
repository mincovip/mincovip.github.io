# 每周动漫追剧日历系统（GitHub Pages 版）

简约大气的「每周动漫追剧日历系统」前端应用：支持星期切换、动漫配置、网址跳转、本地存储、数据导出/导入，并兼容 GitHub Pages 部署。

## 仓库结构

请确保以下文件出现在仓库中（相对路径会用于加载资源）：

- `index.html`
- `css/style.css`
- `js/main.js`
- `.nojekyll`
- `README.md`

## 部署到 GitHub Pages

1. 在 GitHub 新建仓库（例如：`iyf_plan`）。
2. 将上述文件上传到仓库对应路径：
   - `index.html` 放在仓库根目录
   - `css/` 目录下放 `style.css`
   - `js/` 目录下放 `main.js`
   - 根目录放置空的 `.nojekyll`
3. 打开仓库页面：`Settings` -> `Pages`
4. 在 `Build and deployment` 里选择：
   - Source: `Deploy from a branch`
   - Branch: `main`（或你的默认分支）
   - Folder: `/ (root)`
5. 保存后等待部署完成，然后访问 Pages 地址即可。

> 说明：本项目所有静态资源均使用相对路径引用（`./css/...`、`./js/...`），可避免部署后资源加载失败。

## 使用说明

### 星期切换
页面顶部点击 `周一~周日` 按钮即可切换显示内容。

### 动漫配置
点击右上角「动漫配置」，弹窗中可以：
- 添加一条追剧记录（动漫名称 + 播放网址 + 可选备注）
- 编辑已存在的记录
- 删除不需要的记录

### 网址跳转
每条记录在列表中都有「打开」按钮，会在新标签页打开你配置的网址（如未填写网址则按钮不可用）。

### 本地存储
数据会保存在浏览器的 `localStorage` 中。
- 这意味着不会产生跨域请求
- 不依赖服务端

### 数据导出 / 导入
- 「导出数据」会下载一个 JSON 文件，用于备份或迁移
- 「导入数据」选择该 JSON 文件即可恢复数据（导入会覆盖当前数据）

## 常见问题

### 为什么需要 `.nojekyll`？
GitHub Pages 默认会对带下划线的资源做特殊处理。该项目使用 `.nojekyll`（空文件）以避免相关解析问题。

### 资源路径是否会随 GitHub Pages 子路径变化？
本项目静态资源引用使用相对路径，放在仓库根目录部署时通常无需额外配置。
